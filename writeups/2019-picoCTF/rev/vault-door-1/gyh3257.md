# Vault Door 1

## 문제 설명

> This vault uses some complicated arrays! I hope you can make sense of it, special agent.
> The source code for this vault is here: `VaultDoor1.java`.

주어진 Java 소스의 `checkPassword(String password)` 메서드를 분석해 32자 비밀번호를 복원하고, `picoCTF{<password>}` 형식으로 플래그를 제출한다.

## 풀이

### 분석

`checkPassword`는 입력 길이가 32인지 확인한 뒤, `password.charAt(idx) == 'X'` 형태의 단항 비교 31개를 `&&`로 연결한 한 줄짜리 표현식이다.

```java
public boolean checkPassword(String password) {
    return password.length() == 32 &&
           password.charAt(0)  == 'd' &&
           password.charAt(29) == '8' &&
           password.charAt(4)  == 'r' &&
           ...
           password.charAt(31) == '1';
}
```

`charAt` 인덱스 순서가 일부러 뒤섞여 있을 뿐, 각 (index, char) 쌍을 그대로 모으면 정답 문자열이 구성된다. 즉 비밀번호 자체는 단순 평문이며, 단지 소스 코드에서 검증 순서를 셔플한 형태이다.

### 풀이 과정

1. 소스에서 모든 `password.charAt(i) == 'X'` 라인을 추출해 `(i, X)` 페어 리스트를 만든다.
2. 길이 32의 빈 버퍼를 만들고, 각 페어의 위치에 문자를 배치한다.
3. 버퍼를 문자열로 합쳐 `picoCTF{...}`로 감싼다.

```python
pairs = [
    (0,'d'),(29,'8'),(4,'r'),(2,'5'),(23,'r'),(3,'c'),(17,'4'),
    (1,'3'),(7,'b'),(10,'_'),(5,'4'),(9,'3'),(11,'t'),(15,'c'),
    (8,'l'),(12,'H'),(20,'c'),(14,'_'),(6,'m'),(24,'5'),(18,'r'),
    (13,'3'),(19,'4'),(21,'T'),(16,'H'),(27,'3'),(30,'4'),(25,'_'),
    (22,'3'),(28,'f'),(26,'0'),(31,'1'),
]
buf = [''] * 32
for i, c in pairs:
    buf[i] = c
print(f"picoCTF{{{''.join(buf)}}}")
```

## 플래그

```
picoCTF{d35cr4mbl3_tH3_cH4r4cT3r5_03f841}
```

## 배운 점

- 길이 검사 + 인덱스별 단항 비교만으로 비밀번호를 "숨기는" 패턴은 인덱스만 정렬하면 곧바로 평문을 복원할 수 있다.
- 분기 평가 순서를 섞어 두는 것은 정적 분석에 거의 저항력이 없다 — 정상적인 코드 리뷰만으로도 풀리는 수준.
- 동일 패턴이 더 큰 바이너리(strcmp 부분 비교, 광범위한 인라인 검사 등)에서도 자주 보이므로, "인덱스 → 문자" 맵핑을 빠르게 추출하는 스니펫을 도구화해 두면 좋다.
