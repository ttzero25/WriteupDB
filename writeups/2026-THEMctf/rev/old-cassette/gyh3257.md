# Old Cassette

## 문제 설명

> Drop it and it plays by itself.

`main.bin` 파일 하나가 제공된다.

## 풀이

### 분석

`file main.bin` 결과는 `data`지만, 헥스 덤프 시 첫 4바이트가 `00 E0 12 80`으로 시작한다.

**CHIP-8** 프로그램임을 확인:
- `00E0` = CLS (화면 초기화)
- `1280` = JP 0x280 (0x280으로 점프)
- 이후 `A6A6` 패딩 (fill bytes)

파일 크기: 3282 bytes, 로드 베이스: `0x200`

### 프로그램 구조

CHIP-8 디스어셈블 결과:

```
0x200: CLS
0x202: JP 0x280
; 0x204~0x27E: A6A6 padding
0x280: JP 0x900   ; 메인 루프로 점프
```

**메인 루프 (0x900)**:
```
CLS
CALL 0x366    ; (empty stub)
LD VA, 0xa7
LD VB, 0xc3   ; 키 스트림 초기값
LD V0, 1      ; 딜레이 타이머
F0 15         ; 대기
...
XOR VB, V0    ; V0=0 (타이머 만료 시) → VB 불변, VB=0xC3 유지
CALL 0xDB8    ; "THEM?!CTF" 고정 문자 렌더링
```

### 암호화 알고리즘

**키 스텝 함수 (`0x2C0`):**

```
I = 0x800 + VB          ; 256-byte 데이터 테이블 인덱스
V0 = mem[I]             ; 테이블 조회
V0 ^= VB                ; XOR VB
V0 ^= xor_key           ; XOR 키 (VB & 0xC0 기준)
                        ;   0x00 → 0xa9
                        ;   0x40 → 0x5c
                        ;   0x80 → 0xd3
                        ;   0xC0 → 0x76

old_VA, old_VB = VA, VB
VB = (VB + V0) mod 256  ; VB 갱신
VA += carry             ; 올림수 반영
ROL16(old_VA:old_VB, 5) ; 16비트 좌회전 5회
VA ^= rot_high
VB ^= rot_low
```

이후 `mem[0x58B] = VA`, `mem[0x58C] = VB` 저장 (스프라이트 테이블 내 위치).

**키 스트림 PRNG 주기 분석:**
- 초기값: VA=0xa7, VB=0xc3
- tail: 329, period: **34**

**카운터 구조 (`0x282`/`0x294`):**

`CALL 0x282`에 V9, VC, VD, VE를 설정하면 총 키 스텝 수 = `V9 + VC×256 + VD×65536 + VE×16777216`

**스프라이트 테이블 (올바른 베이스):**
| V0 값 | CHIP-8 주소 | 용도 |
|-------|------------|------|
| 0     | 0x400      | 암호화 데이터 |
| 1     | 0x460      | 암호화 데이터 |
| 2     | 0x4C0      | 암호화 데이터 |
| 3     | 0x520      | 암호화 데이터 |
| 4     | 0x600      | 암호화 데이터 |
| 5     | 0x660      | 암호화 데이터 |
| 6     | 0x6C0      | 암호화 데이터 |
| 7     | 0x720      | 암호화 데이터 |

### 복호화 과정

각 문자 복호화:
1. `CALL 0x282`로 키 N스텝 전진
2. `I = sprite_tables[VA & 7] + offset`
3. `char = mem[I] ^ VA ^ VB`
4. `mem[I] = VB` (인-플레이스 업데이트)

**첫 16문자 (0x282 방식):**
| 전진 | 문자 |
|------|------|
| 1    | T    |
| 4    | H    |
| 16   | E    |
| 64   | M    |
| 256  | ?    |
| 1024 | !    |
| ...  | ...  |

**후반 16문자 (0x2AC 방식, V5=0xFF=255마다 리셋):**

각 `CALL 0x2AC` 직전 `LD V5, 0xFF`로 V5가 255로 재설정되어, 매번 `255 × (2^32-1)` 스텝 전진.

`255 × (2^32 - 1) mod 34 = 17` → 사이클 내 위치가 17씩 이동

## 플래그

```
THEM?!CTF{0LD_T4P3_N3V3R_D1E5K7}
```

"K7"는 프랑스어 속어로 "cassette"를 의미 (카세트 = K sept = K7).

## 배운 점

- CHIP-8 에뮬레이션 및 역공학
- PRNG 주기 분석을 통한 대규모 전진 최적화 (Floyd's cycle detection)
- 프로그램 메모리와 코드가 겹치는 자기수정 구조 이해
- 프랑스어 "K7" = cassette 밈
