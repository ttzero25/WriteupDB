## 풀이

### 분석

이 문제는 쿠키와 세션을 이용하여 로그인 상태를 관리한다.
사용자가 로그인에 성공하면 서버는 랜덤한 sessionid를 생성하고, 해당 sessionid와 사용자 정보를 서버의 session storage에 저장한다. 이후 클라이언트는 쿠키에 sessionid를 저장하여 요청 시마다 서버에 전달한다.

서버는 클라이언트로부터 전달받은 sessionid를 통해 사용자를 식별하며, 해당 sessionid에 연결된 username이 admin일 경우 플래그를 출력한다.

### 취약점

`/admin` 경로에서 권한 검증 없이 session storage 전체를 반환한다.
따라서 admin 계정의 sessionid를 획득할 수 있고, 이를 자신의 쿠키 값으로 설정하면 관리자 권한을 획득할 수 있다.

### 익스플로잇

1. guest 계정으로 로그인한다.
2. /admin 페이지에 접속해서 session storage 정보를 확인한다.
3. admin에 해당하는 sessionid 값을 확인하여 내 sessionid 값을 admin sessionid 값으로 바꾼다.
4. admin 계정으로 로그인에 성공하고 플래그 값을 얻을 수 있다.

## 배운 점

특히 세션과 쿠키가 어떻게 연결되어 사용자를 구분하고 웹에서 로그인 상태가 유지되는지 이해할 수 있었다.

또한 서버에서 세션 정보를 제대로 보호하지 않으면, 단순히 쿠키 값을 바꾸는 것만으로도 권한을 우회할 수 있다는 것을 알 수 있었다.
